ФАЗА 1 З 3 📊 Google Sheets + Apps Script
1
Створіть Google Таблицю
  1. Відкрийте sheets.google.com у Google-акаунті цеху
  2. Натисніть «+ Порожня»
  3. Вгорі змініть назву: ZmiyCell Data
✅Таблицю не потрібно заповнювати — скрипт заповнить її сам.
2
Відкрийте редактор Apps Script
  1. У відкритій таблиці: меню Розширення → Apps Script
  2. Відкриється нова вкладка з редактором — файл Code.gs
3
Вставте Code.gs
  1. У редакторі відкрийте Code.gs
  2. Виділіть все: Ctrl+A → видаліть: Delete
  3. Вставте повний вміст файлу Code.gs з цього комплекту
  4. Збережіть: Ctrl+S
⚠️Вставте весь вміст файлу (~300 рядків). Не обрізайте.
4
Запустіть initSheets (один раз)
  1. У рядку вибору функцій (вгорі посередині) оберіть: initSheets
  2. Натисніть ▶ Виконати
  3. Підтвердьте дозволи — «Переглянути дозволи» → оберіть акаунт → якщо буде «Програма не перевірена» → «Розширені» → «Перейти до ZmiyCell» → «Дозволити»
  4. Поверніться у редактор, знову натисніть ▶ Виконати
✅Поверніться в таблицю — з'явилось 8 аркушів: Config, BatteryTypes, Materials, Workers, Tools, Log, RepairLog, PrepItems.
🚫Помилка You do not have permission — переконайтесь, що скрипт прив'язаний до тієї ж таблиці (перевірте URL).
5
Опублікуйте як веб-застосунок → отримайте URL
  1. У редакторі Apps Script натисніть «Розгорнути» (синя кнопка, вгорі праворуч)
  2. Оберіть «Нове розгортання»
  3. Натисніть ⚙ → оберіть тип: «Веб-застосунок»
  4. Налаштуйте:
    • Опис: ZmiyCell API
    • Виконувати від імені: Мене
    • Хто має доступ: Усі
  5. Натисніть «Розгорнути» → підтвердьте дозволи якщо попросить
  6. Скопіюйте URL веб-застосунку — він знадобиться на кроці 9
# URL виглядає так: https://script.google.com/macros/s/AKfycbXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX/exec
⚠️Збережіть цей URL у нотатки — він потрібен на кроці 9. Це ключ до вашої бази даних.
ФАЗА 2 З 3 🐙 GitHub — завантаження коду
6
Встановіть Node.js (якщо ще не є)
  1. Перейдіть: nodejs.org → завантажте LTS версію
  2. Встановіть (все за замовчуванням)
  3. Перевірте — відкрийте командний рядок (Win+R → cmd) і введіть:
node --version # має вивести v20.x.x або вище npm --version # має вивести номер версії
7
Створіть GitHub репозиторій
  1. Зареєструйтесь або увійдіть: github.com
  2. Натисніть «New» або «+» → «New repository»
  3. Назва: zmiycell
  4. Тип: Private
  5. Натисніть «Create repository»
8
Завантажте файли проєкту в GitHub

Структура папки zmiycell/ яку потрібно завантажити:

zmiycell/ ├── api/ │ └── gas.js ← Vercel проксі (вирішує CORS) ├── src/ │ ├── App.jsx ← весь React інтерфейс │ ├── api.js ← клієнтський API │ └── main.jsx ← точка входу ├── index.html ← HTML шаблон ├── package.json ← залежності npm ├── vite.config.js ← конфігурація збірки ├── vercel.json ← маршрутизація Vercel └── Code.gs ← тільки для Apps Script, не завантажувати

Відкрийте командний рядок у папці zmiycell/ і виконайте:

git init git add . git commit -m "ZmiyCell initial commit" # Замініть YOUR_USERNAME на ваш логін GitHub git remote add origin https://github.com/YOUR_USERNAME/zmiycell.git git branch -M main git push -u origin main
💡Якщо git незнайомий — скачайте GitHub Desktop (desktop.github.com). Там все через кнопки: «Add» → «Commit» → «Push».
ФАЗА 3 З 3 🚀 Vercel — деплой і запуск
9
Розгорніть на Vercel
  1. Перейдіть: vercel.com → «Sign Up» → «Continue with GitHub»
  2. На головній сторінці: «Add New…» → «Project»
  3. Знайдіть репозиторій zmiycell → натисніть «Import»
  4. Vercel автоматично виявить Vite — нічого не міняйте у налаштуваннях збірки
  5. Розгорніть розділ «Environment Variables» і додайте одну змінну:
NameValue
GAS_URLВставте URL з кроку 5
https://script.google.com/macros/s/.../exec
  1. Натисніть «Deploy»
  2. Зачекайте ~1-2 хвилини
🎉Vercel видасть посилання виду zmiycell.vercel.app — це і є адреса ZmiyCell для всього цеху!
⚠️Змінна називається GAS_URL (без VITE_). Vercel serverless функції (api/gas.js) читають змінні оточення без префіксу VITE_.
10
Перевірте що все працює
  1. Відкрийте zmiycell.vercel.app
  2. У шапці застосунку має з'явитись ✓ синхр.
  3. Перейдіть у 📦 СКЛАД — мають бути завантажені матеріали
  4. Перейдіть у 👷 КОМАНДА — мають бути 6 стандартних працівників
🚫Якщо статус ✕ помилка — перевірте кроки 5 і 9: URL Apps Script і змінну GAS_URL у Vercel.

Швидка діагностика — відкрийте URL напряму у браузері:

# Відкрийте в браузері — має повернути JSON з даними: https://zmiycell.vercel.app/api/gas?action=loadAll¶ms=[]

Як оновити застосунок у майбутньому

Оновлення React коду (App.jsx та ін.)

  1. Змініть файл
  2. Зробіть git commit + push
  3. Vercel автоматично перезбере і задеплоїть (~1 хв)

Оновлення Code.gs

  1. Відредагуйте код у Apps Script
  2. Розгорнути → Керувати розгортаннями
  3. ✎ → Нова версія → Розгорнути
ℹ️URL застосунку (zmiycell.vercel.app) і URL Apps Script не змінюються при оновленнях — всі посилання залишаються дійсними.
❓ Часті питання
Чи може хтось сторонній отримати доступ до даних?

Застосунок відкритий публічно (без паролю), але URL відомий лише тим кому ви його надіслали. Apps Script URL прихований у Vercel як змінна оточення — користувачі його не бачать.

Скільки коштує?

Google Apps Script і Google Sheets — безкоштовні. Vercel Hobby Plan — безкоштовний для некомерційних проєктів (100GB bandwidth/місяць — для цеху вистачить з надлишком).

Що якщо немає інтернету?

Без інтернету застосунок не працює — дані зберігаються в Google Sheets онлайн. Для автономної роботи потрібна інша архітектура.

Чи можна відкрити з двох телефонів одночасно?

Так. Кожна операція зберігається в Sheets миттєво. Інші пристрої побачать зміни при наступному завантаженні даних або перезавантаженні сторінки.

Як зробити резервну копію?

Дані в Google Sheets — Google автоматично зберігає всю Історію версій (меню Файл → Переглянути історію). Також можна скачати таблицю як .xlsx у будь-який момент.

Помилка «Script function not found: doGet»

Перевірте що в Code.gs є функція doGet, файл збережено, і ви створили нове розгортання після збереження (крок 5).

Apps Script URL змінився після нового розгортання

URL не змінюється при оновленні існуючого розгортання. Якщо ви видалили і створили нове — оновіть змінну GAS_URL у Vercel: Settings → Environment Variables.
